# 三省六部项目文档规范标准

## 1. 总则

### 1.1 目的
本规范旨在建立三省六部项目统一的文档编写标准，确保所有文档具有一致的格式、结构和质量，便于团队协作和知识传承。

### 1.2 适用范围
本规范适用于三省六部项目的所有文档，包括但不限于：
- 项目概述文档
- 架构设计文档
- 流程说明文档
- 用户手册
- API文档
- 操作指南
- 技术规范

## 2. 文档结构规范

### 2.1 标题层级
使用标准Markdown标题层级，从#到####共4级：

```markdown
# 一级标题（文档标题）
## 二级标题（主要章节）
### 三级标题（子章节）
#### 四级标题（细分子节）
```

### 2.2 标准文档结构
所有文档应遵循以下基本结构：

```markdown
# 文档标题

## 1. 概述
[文档目的、范围、背景信息]

## 2. 核心内容
[主体内容，按逻辑分章节]

## 3. 操作流程（如适用）
[操作步骤、流程图等]

## 4. 注意事项
[重要提示、常见问题等]

## 5. 附录（如适用）
[术语表、参考资料等]
```

## 3. 内容编写规范

### 3.1 语言风格
- 使用正式、准确、简洁的语言
- 统一使用中文，标点符号使用中文标点
- 避免口语化表达和模糊词汇
- 语句通顺，逻辑清晰

### 3.2 列表规范
- 无序列表使用 `-` 符号
- 有序列表使用数字加英文句号（如 `1.`）
- 嵌套列表缩进保持一致

### 3.3 代码与示例
- 代码块使用三个反引号包围
- 代码块后需指明编程语言类型
- 代码保持适当的缩进和格式

## 4. 图表规范

### 4.1 ASCII图表
使用标准字符绘制流程图：

```
┌─────────────┐    ┌─────────────┐
│   节点A     │───▶│   节点B     │
│             │    │             │
│ (说明)      │    │ (说明)      │
└─────────────┘    └─────────────┘
```

### 4.2 表格规范
使用Markdown表格格式：

```markdown
| 列1标题 | 列2标题 | 列3标题 |
|--------|--------|--------|
| 内容1  | 内容2  | 内容3  |
| 内容4  | 内容5  | 内容6  |
```

## 5. 文件命名规范

### 5.1 主要文档
- 项目入口文档：`README.md`
- 架构文档：`ARCHITECTURE.md`
- 流程文档：`PROCESS.md`
- 规范文档：`SPECIFICATION.md`

### 5.2 模块文档
- 模块名前缀 + 描述：`MODULE_NAME_DESCRIPTION.md`
- 如：`LIBU_DUTY.md`、`HUBU_RESOURCE_MANAGEMENT.md`

### 5.3 版本文档
- 变更日志：`CHANGELOG.md`
- 版本发布：`RELEASE_NOTES_vX.X.md`

## 6. 引用与链接

### 6.1 内部引用
- 使用相对路径引用项目内其他文档
- 链接文本应明确描述目标内容

### 6.2 外部引用
- 提供完整的URL地址
- 在附录中集中列出所有外部参考资料

## 7. 版本管理

### 7.1 文档版本标识
在文档末尾添加版本信息：

```markdown
---
*文档版本：v1.0*  
*最后更新：YYYY-MM-DD*  
*作者：作者姓名*
```

### 7.2 变更记录
对于重要文档，维护变更日志：

```markdown
## 变更记录

| 版本 | 日期 | 变更内容 | 变更人 |
|------|------|----------|--------|
| v1.0 | YYYY-MM-DD | 初始版本 | 作者 |
| v1.1 | YYYY-MM-DD | 更新XX部分 | 作者 |
```

## 8. 审核流程

### 8.1 文档审核
- 重要技术文档需经过同行评审
- 文档更新需符合三省六部流程
- 重大变更需经门下省审核

### 8.2 质量检查清单
- [ ] 内容准确无误
- [ ] 结构清晰合理
- [ ] 格式符合规范
- [ ] 术语使用统一
- [ ] 链接有效可用

## 9. 模板使用

所有新文档应基于标准模板创建，模板位于 `DOCUMENT_TEMPLATE.md`。
