# 三省六部系统

## 项目概述

三省六部系统是一个基于OpenClaw的分布式任务调度和管理框架，模仿中国古代官僚体系中的三省六部制度，用于协调多个AI代理完成复杂任务。

## 系统架构

### 三省
- **中书省 (Zhongshu)** - 负责政策制定和决策
- **门下省 (Menxia)** - 负责审议和封驳
- **尚书省 (Shangshu)** - 负责执行和实施

### 六部
- **吏部 (Libu_HR)** - 人事管理
- **户部 (Hubu)** - 资源管理  
- **礼部 (Libu)** - 文档、规范、用户界面与对外沟通
- **兵部 (Bingbu)** - 安全和防御
- **刑部 (Xingbu)** - 审计和监督
- **工部 (Gongbu)** - 技术开发和维护

## 核心组件

### 任务ID生成
- 使用 `generate_task_id.py` 脚本生成唯一任务ID
- 格式：`JJC-YYYYMMDD-HHMMSSmmm`
- 确保单调递增，即使进程重启也能保持连续性

### 看板管理系统
- 通过 `kanban_update.py` 脚本管理任务状态
- 支持多种状态转换：Pending, Taizi, Zhongshu, Menxia, Assigned, Review, Done等
- 记录任务流转日志

### 任务调度系统
- 通过 `dispatch_pending_agents.py` 实现自动化任务分发
- 根据任务状态自动分配给相应部门的AI代理
- 包含催办机制，防止任务停滞

## 工作流程

1. **任务创建** - 生成唯一的JJC任务ID
2. **中书省决策** - 制定政策和方向
3. **门下省审议** - 审查和可能的封驳
4. **尚书省执行** - 分配给六部具体执行
5. **状态跟踪** - 通过看板系统实时更新
6. **结果验收** - 完成后归档

## 关键脚本

- `generate_task_id.py` - 生成唯一任务ID
- `kanban_update.py` - 看板状态管理
- `dispatch_pending_agents.py` - 任务自动分发
- `file_lock.py` - 原子文件操作
- `sync_agent_config.py` - 代理配置同步

## 通信协议

各代理间通过标准化的消息格式进行通信，包含任务ID、状态、流转记录等关键信息。

## 使用方法

1. 创建新任务时使用 `generate_task_id.py` 获取唯一ID
2. 通过看板系统跟踪任务状态
3. 各部门代理自动接收分配的任务
4. 定期更新任务状态，确保流程顺畅

## 错误处理和恢复

- 原子文件操作防止数据损坏
- 任务状态监控和催办机制
- 异常状态的自动检测和报告

## 关键技术特点

1. **原子操作**: 使用文件锁保证状态更新的原子性
2. **状态追踪**: 完整的任务生命周期管理
3. **自动分发**: 基于状态的自动任务分配机制
4. **容错机制**: 异常状态恢复和错误处理
5. **可扩展性**: 模块化设计便于功能扩展
